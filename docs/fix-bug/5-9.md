# Bug Fix Record: Wallet Connection & Conflicts (2026-05-09)

## 1. 问题描述
在多钱包共存或特定注入时机下，连接功能出现以下异常：
- **EVM Provider 冲突**：同时安装 MetaMask 和 OKX 时，连接报错 `User rejected` 或 `Provider not found`。
- **Phantom (SOL) 连接失败**：点击 Phantom 按钮报错 `Unexpected error` 或无法拉起弹窗。
- **状态不一致**：连接成功后持久化失效，或切换钱包时状态机发生冲突。

## 2. 核心原因分析
### 2.1 EVM 侧
- **路径抢占**：OKX 等钱包拦截 `window.ethereum` 并伪装标志位。
- **状态机残留**：Wagmi 在未显式断开旧连接时，发起新请求会触发并发冲突。
- **ID 路由模糊**：UI ID 与 Wagmi 内部 Connector ID 缺乏 1:1 映射。

### 2.2 Solana 侧
- **静态列表缺陷**：`SolAdapter` 仅在初始化时读取一次注册表，漏掉了注入较慢的钱包（如 Phantom）。
- **错误信息覆盖**：全局 Catch 块将所有错误强制归类为 EVM 错误，掩盖了真实异常。

## 3. 解决方案

### 3.1 EVM：精准路由与状态清理
- **严苛过滤规则**：在 [evm.ts](file:///Users/chuizi/josen/onchain-coding/snk-wallet-connect-plugin/src/adapters/evm.ts) 中手动解析 `providers` 数组，排除伪装者，锁定纯净 MetaMask；OKX 优先走专属 `window.okxwallet`。
- **强制连接重置**：在 [react.tsx](file:///Users/chuizi/josen/onchain-coding/snk-wallet-connect-plugin/src/react.tsx) 中，切换钱包前强制调用 `wagmiDisconnect` 并增加 100ms 缓冲，彻底清空状态机。
- **锁定 Config 引用**：优化 `useMemo` 依赖，确保 Wagmi 实例在整个生命周期内不被销毁重建。

### 3.2 Solana：响应式适配器
- **动态 Getter**：在 [sol.ts](file:///Users/chuizi/josen/onchain-coding/snk-wallet-connect-plugin/src/adapters/sol.ts) 中将 `wallets` 改为实时 Getter，并暴露 `on('change')` 事件。
- **注册表监听**：在 [react.tsx](file:///Users/chuizi/josen/onchain-coding/snk-wallet-connect-plugin/src/react.tsx) 中实时监听 `wallet-standard` 的 `register` 事件，确保注入较慢的钱包也能被即时捕捉。

### 3.3 错误处理加固
- **命名空间感知**：重构 `normalizeWalletError` 调用，根据 `namespace` 输出精准的错误文案（如 "Failed to connect Solana wallet"）。

## 4. 验证结论
- **多链/多钱包并存测试**：MetaMask、OKX 和 Phantom 现在能稳定地分别拉起对应的插件。
- **状态恢复测试**：刷新页面后 `autoReconnect` 能正确恢复上次连接的钱包。
- **构建状态**：主包与示例工程全量通过。
